# Security Policy

## 安全目標

本專案的安全目標是讓照片分類流程保持本機、可預覽、不可刪除，並避免未經使用者確認的網路傳輸。

## 已實作的防護

- 不提供刪除檔案功能。
- 不產生刪除用途的 PowerShell 指令。
- 執行前顯示 PowerShell 指令供使用者審查。
- 偵測並阻止常見刪除指令，例如 `Remove-Item`、`del`、`erase`、`rm`、`rd`、`rmdir`。
- 偵測並阻止常見網路傳輸指令，例如 `Invoke-WebRequest`、`Invoke-RestMethod`、`curl`、`wget`、`ftp`、`scp`、`ssh`。
- 來源資料夾與輸出資料夾必須是本機路徑，不允許 UNC 網路路徑或網路磁碟。
- 程式執行期間會封鎖 Python socket 連線。
- 程式執行期間會限制外部程序呼叫。

## 限制

這些防護是應用程式層級防護，不等於 Windows 系統層防火牆。若需要更高保證，請在實體斷網環境或受控 Windows 防火牆規則下執行。

## 回報安全問題

若你發現安全問題，請不要公開貼出可利用細節。請先建立私下通報管道，或在 GitHub issue 中只描述影響範圍，不附帶可直接濫用的步驟。
