# Token 使用量紀錄

本文件用來追蹤本專案在 Codex 對話、規劃、實作、測試與文件撰寫中的 token 使用量。

重要限制：目前專案沒有保存過去每一輪的精確 token telemetry，因此既有回合若未由系統提供數字，不應補填成精確值。可用 `未知` 或 `估算` 標示。

## 加速倍率欄位

若某次工作使用了「加速」模式，且該平台或工具明確規定 token 以 `1.5 倍` 計算，請在表格中標記：

- `是否使用加速`：填 `是`、`否` 或 `未知`。
- `加速倍率`：一般填 `1.0`；使用加速且確定以 1.5 倍計算時填 `1.5`；無法確認時填 `未知`。
- `倍率後總 tokens`：若有精確總 tokens，計算方式為 `總 tokens × 加速倍率`；若總 tokens 或倍率未知，填 `未知`。
- `倍率可信度`：填 `精確`、`估算` 或 `未知`。

不要在沒有證據時假設使用了加速，也不要把估算倍率寫成精確值。

## 填寫規則

- `日期時間`：使用本機時間，格式建議為 `YYYY-MM-DD HH:mm`。
- `工作階段`：簡短描述本次工作，例如 `GUI 修正`、`安全檢查`、`README 更新`。
- `輸入 tokens`：使用者訊息、系統上下文、讀取檔案摘要等輸入消耗。
- `輸出 tokens`：模型回覆、程式碼、文件內容等輸出消耗。
- `工具/檔案讀取 tokens`：大量讀檔、搜尋結果、測試輸出等可另列。
- `總 tokens`：若有精確數字，填入總和；若沒有，填 `未知`。
- `是否使用加速`：若本輪使用加速模式，填 `是`；若未使用，填 `否`；無法確認填 `未知`。
- `加速倍率`：未使用加速填 `1.0`；確認加速以 1.5 倍計算填 `1.5`；無法確認填 `未知`。
- `倍率後總 tokens`：用於估算加速後的計費或消耗權重。
- `可信度`：填 `精確`、`估算` 或 `未知`。
- `備註`：記錄本輪主要成果或限制。

## 維護方式

1. 每完成一個明確工作節點後新增一列，例如 `安全功能修改`、`GUI 調整`、`測試修正`。
2. 若 Codex 或外部工具提供精確 token 數，填入數字並標 `精確`。
3. 若只是依工作量粗估，務必標 `估算`。
4. 若沒有數據，不要猜成精確值，填 `未知`。
5. 若確認使用加速且規則為 1.5 倍，填 `是否使用加速=是`、`加速倍率=1.5`。
6. 每次準備提交 Git 前，檢查本文件是否需要補一列。
